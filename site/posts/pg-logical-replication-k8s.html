Postgres logical replication on k8s, the bits no one mentions.

postgres k8s replication

WAL sender timeouts, publication identity columns, and that one quirk with sequences after failover. Here's everything I wish the docs told me upfront.

The setup

Two postgres 16 instances running in a k8s cluster. Publisher on one statefulset, subscriber on another. Sounds simple. It isn't.

WAL sender timeout

The default wal_sender_timeout is 60s. In k8s, where pod restarts and network blips are normal, this is too aggressive. I bumped it to 300s and the replication stopped dropping every time a node got rescheduled. 

ALTER SYSTEM SET wal_sender_timeout = '300s';
SELECT pg_reload_conf();

Publication identity columns

If your table doesn't have a primary key, logical replication will refuse to replicate UPDATE and DELETE operations. You need to set a replica identity. 

ALTER TABLE my_table REPLICA IDENTITY FULL;

The sequence gotcha

Sequences are NOT replicated. After a failover to the subscriber, your sequences will still be at their old values. You need to manually reset them.

Always script your sequence resets as part of your failover runbook.
